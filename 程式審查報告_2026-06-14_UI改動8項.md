## 三、A–E 維度小結

**A. 正確性** — 全數實測通過。圓餅角度/百分比/`large-arc-flag`、單類別 `<circle>` 邊界、跨幣別台幣換算、同日新增先後、日期升冪、`moveTxn` 邊界、`commitManualOrder` 篩選子集合併、四維度（type×日期×排序×新分頁）疊加 — 無正確性缺陷。

**B. 安全（XSS）** — 所有**自由文字**輸入（item / note / 帳戶名 / 分類）在主清單與新分頁快照皆經 `escapeHtml`，實測逸出正確；`openChartTab` 的 `document.write` 僅注入數字與已逸出文字，安全。殘留低風險注入面（**僅能透過竄改匯入 JSON 觸發＝self-XSS**，正常 UI 不可達）：`onclick` 內 `${t.id}`/`${d}`（系統產生的 id 與 `type=date` 值）、`style="background:${a.color}"`（顏色由 PALETTE 指派）。其中 `style` 的 `${a.color}` 為既有寫法（非本批引入）。建議補 `sanitizeColor()` 與匯入 schema 驗證（§四 P2）。

**C. 邊界** — 空資料 / 單筆 / 全 topup / 日期刪光自動回全部 / 自訂模式單筆 ▲▼ no-op — 實測皆穩。`commitManualOrder` 空陣列邊界經驗證不損資料。

**D. 行動裝置（Android Chrome）** — `⠿` 把手 `touch-action:none` 與列原生捲動分離（正確）；膠囊列原生橫向捲動可用；圖示鈕 32px 觸控達 AA；`#ff5a36` 深底對比足夠；本批未動 PWA meta，**「加入主畫面」不受影響**。注意：Android 完整安裝（A2HS 提示）本就因**無 `manifest.json` / service worker** 而受限 — 此為既有狀態，非本批造成。

**E. 一致性 / 資料完整** — 自訂排序改動 `data.txns` 順序後，匯出 JSON/CSV、各帳戶統計、圓餅計算實測**不受順序影響**；`localStorage` key `jp_trip_ledger_v2` 結構未破壞，JSON round-trip 穩定。

## 四、總分 9/10 — 扣掉的 1 分如何補回

所有缺項皆 Low；以下依「趁手就修」與「可排後續」分類。

### 趁手修（≈ 3 行，建議結案前順手做）
- **P1-a** `.move-btn { height:22px }` → `height:24px`（達 WCAG 2.5.8 AA 觸控目標）。
- **P1-b** `pieSVG` 的 `<svg class="pie"` → 加 `role="img" aria-label="消費分佈圓餅圖"`（螢幕閱讀器可辨識）。
- **P1-c** `txnTWDValue` 首行加 `if (t.type !== 'expense') return 0;`（金額排序時轉帳沉底，消除語意混淆）。

### 可排後續（P2，不阻擋結案）
- **P2-a** 死 CSS 清除：`.bar-row` / `.bar-track` / `.bar-fill`（行 65–70、171）改用圓餅後已無 JS 引用。
- **P2-b** `enableDragScroll`：window listener 改用 `AbortController` 或 pointer events；`mouseup` 時一併重設 `moved`。
- **P2-c** `openChartTab` 改 Blob URL + `noopener`，取代 deprecated 的 `document.write`。
- **P2-d** `enableRowReorder` 的 `end()` 加 `if(!listEl.isConnected) return;`（防拖曳中重繪的靜默無效）。
- **P2-e** 拖曳邊緣自動捲動（長清單體驗；旅遊記帳通常 ≤100 筆，影響有限）。
- **P2-f** 匯入 JSON 加 schema/型別驗證 + `sanitizeColor()`，關閉 self-XSS 殘面。
- **P2-g**（既有，非本批）`load()` 對 `data.rate` 加 `if(typeof d.rate!=='number'||d.rate<=0) d.rate=4.98;`，避免 0/NaN 時 `toTWD` 靜默誤算。
